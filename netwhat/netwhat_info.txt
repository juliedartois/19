Méthode détailler (adresse réseau/ broadcast)
    ETAPE 1 : On passe l adresse décimale en binaire

        exemple :   192.168.0.15/255.255.255.240
            masque  255.255.255.240 =>  1111 1111.1111 1111.1111 1111.1111 0000
            adresse 192.168.0.15    =>  1100 0000.1010 1000.0000 0000.0000 1111

    ETAPE 2 : On separe la partie RESEAU | HOTE definie par le masque

            masque  255.255.255.240 =>  1111 1111.1111 1111.1111 1111.1111 | 0000
            adresse 192.168.0.15    =>  1100 0000.1010 1000.0000 0000.0000 | 1111

    ETAPE 3 :   Pour obtenir l adresse RESEAU on met tous les bits de la partie hôte a 0

            adresse réseau    =>  1100 0000.1010 1000.0000 0000.0000 | 0000
            en decimal => 192.168.0.0

                Pour obtenir l adresse broadcast on met tous les bits de la partie hôte a 1
            adresse broadcast = > 1100 0000.1010 1000.0000 0000.0000 | 1111
            en decimal => 192.168.0.15

    ETAPE 4 : 
            si l adresse ip = adresse réseau => celle ci est donc l adresse réseau
            si l adresse ip = adresse broadcast => celle ci est donc l adresse de broadcast
            si l adresse ip est compris entre le réseau et le broadcast => celle ci est une adresse machine

Calculs de sous réseau
    On cherche n tels que 2^n = x
    avec x : le nombre de sous réseaux 

    exemple : 215.210.192.0 / 255.255.240.0
    masque  : 1111 1111.1111 1111.1111 | 0000.0000 0000
    adresse : 1101 0111.1101 0010.1100 | 0000.0000 0000
    
    on souhaite 2 sous réseau
        2^0 = 1 trop petit
        2^1 = 2 ok
    n = 1 donc

    RRRR RRRR.RRRR RRRR.RRRR SHHH.HHHH HHHH
    1101 0111.1101 0010.1100 1000.0000 0000  => 255.255.248.0
    il reste dnc 11 bits a l hote
        2^11 - 2 = 2048 -2 = 2044 
    il y a donc 2044 nombre d hôte utilisable.
    adresse réseau    = 1101 0111.1101 0010.1100 1|000.0000 0000 => 215.210.192.0
    adresse broadcast = 1101 0111.1101 0010.1100 1|111.1111 1111 => 215.210.199.255
